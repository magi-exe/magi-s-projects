rocket_launchGetting Started

1

Install XAMPP

Download and install XAMPP from apachefriends.org. Start Apache and MySQL from the Control Panel. These must be running every time you use the library system.

2

Import the Database

Open http://localhost/phpmyadmin in your browser. Click Import, choose the schema.sql file from the comboni-v4 folder, and click Go. This creates the comboni_library database.

3

Place the Files

Copy the comboni-v4 folder into C:\xampp\htdocs\ (Windows) or /Applications/XAMPP/htdocs/ (Mac).

4

Open the App

In your browser, go to http://localhost/comboni-v4/. You will be taken to the login page. Never double-click the HTML files directly — they must run through Apache.

5

Add Optional Assets

For icons and fonts to display offline, download and place these in assets/fonts/: material-icons.woff2, NotoSansEthiopic woff2 files. For charts, place chart.umd.min.js in assets/js/. See README for download links.

lockLogin & Access

Access Levels

  • Admin: Full access — add, edit, delete, return, change settings
  • Guest (View Recent Activity): Can only view the last 15 borrow records. Cannot make any changes.
1

Admin Login

Enter the admin password and click "Enter System". Default password: Comboni-library5634. Change it immediately in Settings → Change Admin Password.

2

Guest Access

Click "View Recent Activity" without entering a password. You will be able to see the homepage activity feed only. All edit buttons are hidden.

add_circleAdding a Borrow Record

1

Choose Mode

At the top of the home form, click Teacher or Student depending on who is borrowing. The form fields adapt to the selection.

2

Select Borrower

For teachers, start typing the teacher's name — a dropdown will appear. You must select from the dropdown; free-text is not accepted for teachers. For students, type the name freely — the system learns names and shows suggestions.

3

Select Book

Start typing the book title. Select from the dropdown to auto-fill the book code. You can also type the code manually. If the book doesn't exist yet, click "+ Add new book".

4

Save

Click Save. The record appears in the Recent Activity feed on the right immediately.

library_booksBulk Borrow

Use this when one person borrows multiple books at once — e.g. a teacher taking 10 books for a class project.

1

Switch to Bulk Mode

Click the Bulk Borrow tab at the top of the home form.

2

Set Borrower

Type the borrower's name and select Teacher or Student. For teachers, select from the dropdown.

3

Add Book Rows

Each row represents one book. Type the title (autocomplete available) and the code. Click + Add Book Row to add more rows. Remove rows with the × button.

4

Save All

Click Save All Borrows. All rows are saved as separate borrow records with the same borrower and date.

undoReturning Books

Three ways to mark a book returned

  • Activity feed: Click the ↩ button on a "Taken" card in the home activity feed
  • Teachers' Log: Find the record in the teacher's table and click "Mark Returned"
  • Students' Log: Find the record in the student's table and click "Mark Returned"
1

Accidentally marked returned?

Every "Returned" record has a Mark Taken button that reverses the status back to "Taken". Find it in the Teachers' or Students' Log.

menu_bookBooks Inventory

The Books page maintains the library's physical collection — separate from borrow records.

1

Add Books Manually

Click Add Book and fill in the title, code (must be unique), author, category, shelf location, and quantity.

2

Import from CSV

Click Import CSV. Prepare a CSV file (see CSV Rules section). Books with duplicate codes are skipped automatically.

3

Availability Bar

Each book card shows a colored bar indicating how many copies are available vs. currently borrowed. Red = all copies out, green = copies available.

ruleCSV Rules

Borrow Records CSV (Export / Import)

  • First row must be the header row with column names
  • Required columns: borrower_name, borrower_type, book_title, book_code, date_taken
  • Optional columns: status, return_date, notes, return_notes
  • borrower_type must be exactly teacher or student
  • date_taken format: YYYY-MM-DD HH:MM:SS
  • status must be taken or returned
  • Lines beginning with # are treated as comments and skipped
  • Duplicate detection: a record is a duplicate if it has the same book_code + date_taken + borrower_name

Books Inventory CSV (Books page)

  • Required columns: title, code
  • Optional: author, isbn, category, shelf_location, quantity, description
  • Books with duplicate codes are skipped — existing records are not overwritten
  • Quantity defaults to 1 if not provided

Tip: Export an existing dataset first to see the exact expected format.

schoolTeachers' Log

1

View Teacher Cards

The page shows a card for each teacher with their current borrow count and total borrows. Click a card to expand their full borrow history below.

2

Filter & Sort

Use the filter bar above the table to filter by status, date range, or book. Click column headers to sort. Hold Shift and click a second column for multi-level sorting.

3

Bulk Actions

Check the checkboxes on rows to select multiple records. A blue action bar appears — use it to bulk mark returned or export selected records.

4

View Audit History

Click the audit icon on any row to see the complete edit history — who changed what, and when.

groupsStudents' Log

1

Search Students

Use the search bar on the left to find a student. Frequent borrowers appear by default. Click a name to see their borrow history.

2

Frequency Badges

A badge next to the student name shows how many total borrows they have — useful for identifying heavy readers.

bar_chartReports

The Reports page requires chart.umd.min.js in assets/js/ for charts to display.

1

Summary Cards

Top row shows at-a-glance numbers: total borrows, currently out, returned, overdue count, number of teachers, and students in the system.

2

Charts

Daily trend (30 days), monthly trend (12 months), top 10 most borrowed books, most active borrowers, and overdue breakdown by class.

archiveArchive

1

How Archiving Works

Records older than the archive period (configured in Settings, default 30 days) are automatically moved here by the cron job. They no longer appear in the active log but are preserved.

2

Restore a Record

Click Restore on any archived record to move it back to the active borrow log. It will appear again in Teachers' / Students' logs.

3

Manual Archive

Click Run Archive Now to immediately archive all eligible records without waiting for the scheduled cron job.

import_exportExport / Import

1

Export

Use the filter options to choose which records to export. Click Export Visible Dataset to export only filtered records, or Export All for everything. Files are saved as CSV with a metadata header row.

2

Import

Drag a CSV file onto the import zone or click to browse. The system validates the file and shows a summary before applying. Choose to skip or overwrite duplicates.

3

Backup Strategy

Export all records weekly as a backup. Store the CSV files somewhere safe. You can re-import them at any time to restore data.

settingsSettings

1

Change Password

Go to Settings → Change Admin Password. Enter your current password and the new one. Minimum 6 characters.

2

Switch Language

Click 🇬🇧 English or 🇪🇹 አማርኛ to switch the interface language. Requires Noto Sans Ethiopic font for Amharic to display correctly.

3

Return Period

Set how many days before a borrowed book is considered overdue. Default is 14 days. Affects overdue badges and reports.
